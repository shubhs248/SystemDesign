# Part 3 — DevOps / Infra Design (your differentiator)

## 🎯 Goal
The classic prompts (Part 2) test general system design. **These** test what a DevOps/SRE/Platform engineer is actually hired to design: pipelines, observability, and resilient infrastructure. Nailing these is what sets your interview apart from a generic SWE candidate.

The same [framework](../00-the-framework/README.md) applies — clarify, estimate, design, deep-dive, scale & harden — but the vocabulary shifts to **reliability, deployment safety, automation, and cost**.

## 📝 How to use each scenario
1. Read the `scenario-*.md` prompt and requirements.
2. **Design it yourself first** (~20 min, out loud).
3. Compare with the worked answer in [`solutions/`](solutions).

| # | Scenario | Core idea it teaches |
|---|----------|----------------------|
| 1 | [CI/CD Pipeline](scenario-1-cicd-pipeline.md) | build→test→deploy, deployment strategies, rollback, security |
| 2 | [Observability Platform](scenario-2-observability-platform.md) | metrics/logs/traces at scale, alerting, retention/cost |
| 3 | [Highly Available, Scalable Web App](scenario-3-ha-scalable-web-app.md) | multi-AZ HA, autoscaling, no SPOF, DR |

➡️ Then test your concepts: [INTERVIEW-QUESTIONS.md](../INTERVIEW-QUESTIONS.md).
